Self-correcting Code-adaptive DevOps Agent

I built a self-correcting code-adaptive DevOps Agent that takes in Resource Provider (RP) specific files, generates output code, attempts to build the project, feeds build errors back to the model until errors are fixed, and finally pushes code to Azure DevOps with an automated PR.

UI Development & User Experience

The UI was built using Streamlit for a lightweight framework as the work was LLM-heavy. The design evolved from a simple chatbot to include:

  • Text boxes and checkboxes to select files needed for uploading into LLM memory
  • Real-time display of generated code on the UI
  • Build error visualization and tracking
  • Pipeline re-running interface with build error feedback
  • Corrected code regeneration display
  • PR creation button with final PR link display

Technical Architecture & Azure Integration

The project began with exploring Azure services and deciding to use Azure Foundry SDK to handle tools, webhooks, and integrations. While a simple RAG pipeline using the SDK worked initially, the code quality wasn't sufficient and executed with build errors. This led to implementing three major improvements:

1. Extensive Prompt Engineering

  • Developed sophisticated prompt engineering strategies with few-shot examples of specific code patterns
  • Helped generate the correct skeleton and structure of the code
  • Significantly improved initial code generation quality

2. Metadata-Based Code-Adaptive RAG Approach (Key Innovation)

Problem: Initially, single files were uploaded to the vector database and retrieved based on highest match to new RP descriptions. This caused significant information mismatches and incomplete project context.

Solution: I invented a metadata-based approach to retrieve relevant projects as complete units:

  • Created metadata structure containing: project name, RP description and functionality, and names of all files present in the RP
  • Only the description was embedded in the vector database, not individual files
  • For incoming new RP requests, the new description was embedded and matched against existing RP descriptions
  • Most similar RPs were retrieved as whole projects rather than individual files
  • This approach made the RAG pipeline code-adaptive rather than just text-adaptive
  • Significantly improved performance by maintaining project structure and context

3. Self-Correcting Build Automation (Major Breakthrough)

Challenge: We were manually building projects and feeding errors backâ€”a time-consuming and inefficient process.

Innovation: Automated the entire build-fix-rebuild cycle:

  • Wrote a script to automatically build the project after code generation
  • Captured build errors and fed them back to the LLM automatically
  • Provided new instructions along with build errors to guide regeneration
  • LLM regenerated the entire codebase with corrections
  • Loop continued until the build was successful
  • Result: Improved code writing time by 75%

Key Impact & Results

  • 75% improvement in code generation accuracy and development time
  • Reduced manual intervention through automated build-fix cycles
  • Invented metadata-based code-adaptive RAG approach for maintaining project context
  • Successfully integrated with Azure DevOps for automated PR creation
  • Deployed as production-ready customer-facing application
